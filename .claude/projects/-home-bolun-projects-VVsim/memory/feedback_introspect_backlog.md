---
name: Introspect backlog report format
description: How to report the backlog section at the end of /introspect
type: feedback
---

Do NOT list the top 3 existing backlog tasks by criticality — user can check tasks directly.

Instead, report the top 3 suggestions that came up during the session but didn't make the logging cut. Explain why each didn't meet the bar (low value, low urgency, style nit, etc.).

**Why:** The existing task list is visible to the user on demand; repeating it wastes space and adds no value.

**How to apply:** In Step 8 of /introspect, replace the "top 3 by criticality from the full backlog" block with "top 3 near-miss candidates from this session and why they weren't logged."
